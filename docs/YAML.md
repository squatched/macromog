# YAML Reference

Macromog uses two YAML files: the **macro file** (exported and imported) and
**`config.yml`** (runtime configuration). This document describes every field
in both schemas.

---

## Macro file

Produced by `macromog export` and `//mmog export`; consumed by `macromog import`
and `//mmog import`.

Exports are **sparse** — only books, sets, and macros that contain data are
included. Templates (generated by `macromog template`) are **dense** — every slot
within the requested scope is present with empty name and content fields, ready to
fill in. Use `macromog template` to get a valid skeleton rather than constructing
one by hand.

### Annotated schema

```yaml
version: 1                         # Required. Always 1.

character: squatched               # Optional. Character name embedded at export
                                   # time for reference; not used on import.

exported_at: "2026-06-20T03:30:00Z"  # Optional. ISO 8601 export timestamp.
                                      # Omitted in template files.

scope:                             # Required. Declares which portion of the
                                   # macro library this file has authority over.
  level: full                      # full | book | set | macro
  selections:                      # Omitted when level is full.
    - {book: 1}                              # book-level entry
    - {book: 1, set: 2}                      # set-level entry
    - {book: 1, set: 2, type: ctrl}          # set-level, ctrl keys only
    - {book: 1, set: 3, type: alt, key: 1}   # macro-level entry

books:
  1:                               # Book index. Integer, 1–40.
    name: WHM75NIN                 # Optional. Custom book name displayed in-game.
                                   # Max 15 chars. Alphanumeric only — no spaces,
                                   # symbols, or punctuation. Omit or leave empty
                                   # for an unnamed book.
    sets:
      1:                           # Set index. Integer, 1–10.
        header_unknown: 1234567890 # Optional. Bytes 4–7 of the DAT set header;
                                   # purpose unknown. Preserved on round-trip so
                                   # re-imports don't alter what the game wrote.
                                   # Write 0 (or omit) when generating by hand.
        ctrl:                      # Ctrl-key macros for this set.
          1:                       # Key index. Integers 1–9, then 0 (see Indexing).
            name: Cure             # Optional. Button label shown in-game.
                                   # Max 8 bytes. CJK characters (Shift-JIS) cost
                                   # 2 bytes each, so Japanese titles cap at 4 chars.
            contents:              # Macro lines. 0–6 entries.
              - /ma "Cure IV" <t>  # Each line: max 60 chars. Lines containing
              - /echo done         # auto-translate markers are exempt (see below).
        alt:                       # Alt-key macros for this set. Same structure as ctrl.
          1:
            name: Buffs
            contents:
              - /ma "Protect IV" <me>
  5:                               # Books are sparse — gaps are fine.
    name: RDM75NIN
    sets:
      3:
        ctrl:
          0:                       # Key index 0 = the "10" slot in-game (rightmost).
            name: Refresh
            contents:
              - /ja "Refresh" <me>
```

### Constraints

| Item | Limit | Notes |
|------|-------|-------|
| Books | 1–40 | Indices 1–40 |
| Book name | 15 chars | Alphanumeric only. No spaces, symbols, or punctuation. |
| Sets per book | 1–10 | Indices 1–10 |
| Macros per set | 20 | 10 ctrl + 10 alt |
| Macro key indices | 1–9, then 0 | `0` is the tenth slot (rightmost in-game), not the first |
| Macro name | 8 bytes | Any printable character. CJK chars (Shift-JIS) cost 2 bytes — Japanese titles cap at 4 chars |
| Lines per macro | 6 | — |
| Chars per line | 60 | Printable only; no tabs, newlines, or control codes. Lines with auto-translate markers are exempt. |

### Indexing

All indices are **1-based** except macro keys, which follow keyboard layout order:
`1, 2, 3, 4, 5, 6, 7, 8, 9, 0`. The key index `0` represents the tenth slot
(displayed as "10" in some contexts); the integer `10` is never used as an index.

### Extension token markers

FFXI embeds binary extension tokens inside macro text for auto-translate phrases,
elemental symbols, and other special in-game text. On export these tokens are
converted to printable placeholder strings wrapped in `≺` (U+227A) and `≻`
(U+227B) — the mathematical "precedes" and "succeeds" symbols, chosen to avoid
collisions with ordinary macro text. On import they are converted back to binary.

These characters are **not** the standard ASCII `<` and `>`. A macro line
containing `<[12345678]>` (typed by a user) is treated as literal text; only
lines containing the exact Unicode delimiters trigger special handling.

#### Marker types

| Marker | Binary | What it represents |
|--------|--------|-------------------|
| `≺[XXXXXXXX]≻` | `0xFD <4-byte-id> 0xFD` | Auto-translate phrase, spell, or item reference. `XXXXXXXX` is the 8-digit hex resource ID. |
| `≺element:N≻` | `0xEF 0x1F–0x26` | Elemental icon (N=0–7: fire, ice, wind, earth, lightning, water, light, dark). |
| `≺autotrans:start≻` / `≺autotrans:end≻` | `0xEF 0x27` / `0xEF 0x28` | The visual `≺`/`≻` decorators the game displays around auto-translate phrases in chat. Rarely used in macros directly. |
| `≺byte:XX≻` / `≺byte:XXYY≻` | single unrecognized byte / undecodable Shift-JIS pair | A byte or pair the decoder does not recognize — preserved verbatim. |

Example in YAML:

```yaml
contents:
  - /ma ≺[07021203]≻ <me>
  - ≺[02020114]≻ good luck!
```

#### Line-length exemption

Only resource markers (`≺[XXXXXXXX]≻`, regex `≺\[[0-9A-Fa-f]{8}]≻`) exempt a
line from the 60-character check. The other types (`element`, `autotrans`, `byte`)
do **not**. Their placeholder strings are also longer than their binary form —
`≺element:0≻` is 12 characters in YAML but 2 bytes in the DAT — so for lines
containing those markers macromog's check may be stricter than the game's actual
limit. In practice elemental symbols and autotrans decorators appear infrequently
in macros, so this rarely matters.

#### Authoring limitations

There is no way to author a legitimate, new resource marker from scratch without
first exporting a macro that already contains the desired auto-translate phrase.

Any YAML string matching `≺[XXXXXXXX]≻` is treated as a resource marker on
import. Since `≺` and `≻` are not standard keyboard characters this cannot be
triggered by accident, but it does mean a string that looks like a marker cannot
be stored as literal macro text.

### Scope field

The `scope` field embedded at export time tells `import` exactly what it may
alter — and what it must leave alone.

| `level` | Import authority | Absent entries |
|---------|-----------------|----------------|
| `full` | All 40 books | Absent books are deleted |
| `book` | Only the listed books | Absent sets within those books are deleted |
| `set` | Only the listed (book, set) pairs | Absent macro slots within those sets are zeroed |
| `macro` | Only the listed individual slots | Nothing is deleted |

For the `--scope` flag syntax used at export and import time, see
[CLI.md — Scope Selectors](CLI.md#scope-selectors).

## config.yml

Created automatically on first use. Location:

| Context | Path |
|---------|------|
| Linux | `~/.config/macromog/config.yml` |
| Windows | `%APPDATA%\macromog\config.yml` |
| Wine (Linux home mapped) | `~/.config/macromog/config.yml` (shared with the in-game addon) |
| Override | `MACROMOG_CONFIG` environment variable — absolute path to config file |

### Annotated schema

```yaml
version: 1                         # Required. Always 1.

preferences:
  default_offering: true           # Optional. Controls whether macromog shows
                                   # tips about setting defaults at selection
                                   # prompts. Absent = true. See CLI.md.

default_install: steam             # Optional. Name of the install used when
                                   # no --install flag is given and multiple
                                   # installs are registered.

installs:
  steam:                           # Install name — a label you choose. It has no
                                   # special meaning to macromog; it exists solely
                                   # so you can refer to this install with
                                   # --install <name> instead of typing the full
                                   # path. Short, lowercase, no spaces recommended.
    path: /home/you/.steam/steamapps/compatdata/230330/pfx/drive_c/Program Files (x86)/PlayOnline/SquareEnix/FINAL FANTASY XI
                                   # Required. Absolute POSIX path to the FFXI
                                   # root (the directory containing USER/).
                                   # Always stored as a POSIX path even under
                                   # Wine — never stored as C:\... .
    default_character: a1b2c3d4    # Optional. Folder ID of the default character
                                   # for this install. Set automatically when you
                                   # register the first alias.
    characters:                    # Optional. Omitted when no aliases exist.
      a1b2c3d4:                    # Folder ID as FFXI stores it — 8 hex chars.
        name: Squatched            # Friendly alias used with --char-name.

  lutris:
    path: /home/you/Games/ffxi/drive_c/Program Files (x86)/PlayOnline/SquareEnix/FINAL FANTASY XI
    default_character: a1b2c3d4
    characters:
      a1b2c3d4:
        name: Squatched            # Same name is fine — aliases are scoped per install.
```

### Field reference

| Field | Type | Notes |
|-------|------|-------|
| `version` | integer | Always `1` |
| `default_install` | string | Name key from `installs`; omitted when unset |
| `installs.<name>.path` | string | Absolute POSIX path to FFXI root; never `C:\...` |
| `installs.<name>.default_character` | string | 8-char hex folder ID; omitted when unset |
| `installs.<name>.characters.<id>.name` | string | Friendly alias; case-insensitive; unique within an install |
| `preferences.default_offering` | boolean | `true` or `false`; absent = `true` |

**Paths** are stored as absolute, normalized POSIX paths: `~` expanded, trailing
slashes removed, symlinks preserved as written.

For how `config.yml` is used at runtime — install resolution order, character
selection, interactive prompts, and the `config` subcommand — see
[CLI.md — Configuration](CLI.md#configuration).
